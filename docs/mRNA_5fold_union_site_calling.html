feat_levels <- c("5UTR", "CDS", "3UTR", "ncRNA", "Intron", "Intergenic")
N_FOLDS <- 5

Strategy

Instead of calling pU sites once from all cell lines pooled, we partition the 51 cell lines into 5 disjoint folds (~20% each; seed = 2026, so the partition is reproducible), joint-call sites within each fold (pool that fold’s ~10 cell lines’ reads, one chi-squared test per site, FDR < 0.01, rate_diff > 0.05, depth >= 20), then combine across folds and count how many of the 5 folds each site was called in.

This turns the number of folds into a reproducibility filter: keeping sites called in >= k of 5 folds. k = 1 is the plain union (any fold); higher k requires a site to reproduce across independent 20% subsets. The motivation: the earlier discovery/replication analysis showed only ~40-60% of sites called from a single 20% pool replicate, so a plain union carries a lot of non-reproducible sites — requiring >= k folds should filter them.

Threshold curve: site count, feature mix, and BACS overlap vs. >= k folds

thr <- rbindlist(lapply(1:N_FOLDS, function(k) {
  sub <- sites[n_folds >= k]
  data.table(
    min_folds             = k,
    n_sites               = nrow(sub),
    pct_cds_utr           = round(100 * mean(sub$feature %in% c("CDS", "5UTR", "3UTR")), 1),
    pct_intron_intergenic = round(100 * mean(sub$feature %in% c("Intron", "Intergenic")), 1),
    n_bacs                = sum(sub$in_bacs),
    pct_bacs              = round(100 * mean(sub$in_bacs), 1)
  )
}))

Findings

  • Reproducibility strongly enriches for validated (BACS) sites. BACS overlap rises from 3.7% (plain union) to 16.2% (all 5 folds) — a 4.4× enrichment. The single-fold-only sites are nearly devoid of BACS overlap (63 BACS sites out of 7774 single-fold sites), i.e. they are mostly spurious — the fold filter removes them effectively. A reproducibility filter is well justified.
  • It does NOT enrich for CDS/UTR — it slightly de-enriches. CDS+UTR fraction drops (63.3% → 54.3%) and intron+intergenic rises as more folds are required. So the genomic feature distribution is not a good “realness” discriminator here; BACS overlap is the trustworthy signal, and it says reproducibility works even though the feature mix does not shift as one might expect.

Recommendation

  • Avoid the plain union (>= 1) — 3.7% BACS and dominated by non-reproducible single-fold sites.
  • >= 2 folds is the natural minimum (analogous to the existing n_called >= 2): it roughly doubles BACS enrichment (8.3%) while keeping 4,888 sites — the largest single jump in quality.
  • >= 5 folds for maximum purity (16.2% BACS) at the cost of dropping to 1,876 sites.

The >= 2 to >= 5 range is the precision-vs-sensitivity choice. Note this used the stricter FDR < 0.01 calling filter, so even >= 2 is fairly clean.
