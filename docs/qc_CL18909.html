  • Sensitivity of site calls to depth and ratio thresholds

Sensitivity Analysis: Depth and Ratio Cutoffs

The initial filter only removes sites present in the input. Further thresholding on read depth and C/(C+T) ratio in the treated sample is needed to improve specificity. This section explores how many sites survive at different combinations of these two cutoffs.

  • Minimum depth: requires a minimum number of reads covering a site in the treated sample. Low-depth sites have unreliable ratio estimates.
  • Minimum ratio: requires a minimum fraction of reads showing the C mismatch. A ratio of 0 means all reads show T (no modification), while a ratio of 1 means all reads show C (complete modification or stop).

The figure below shows a heatmap for each sample where each cell reports the absolute number of sites retained and the percentage of post-filter sites they represent. Cells towards the top-right (high depth AND high ratio) are the most stringent; cells at the bottom-left (low depth, any ratio) are the most permissive.

What to look for: - Consistent patterns across samples indicate reproducible signal. - A ratio cutoff of ~0.1 and depth >= 20 are the default cutoffs from Yanming’s pipeline.

Per-sample sensitivity analysis

cutoff_all <- map_dfr(all_samples, function(samp) {
  dat <- all_filter %>% filter(sample_id == samp)
  expand.grid(min_depth = DEPTH_CUTOFFS, min_ratio = RATIO_CUTOFFS) %>%
    mutate(
      sample_id    = samp,
      n_sites      = mapply(function(d, r)
        sum(dat$depth >= d & dat$ratio >= r, na.rm = TRUE),
        min_depth, min_ratio),
      pct_retained = n_sites / nrow(dat) * 100
    )
})

Pooled sensitivity analysis

In addition to per-sample cutoffs, it is useful to examine how many unique genomic positions survive across all samples combined. Here all post-filter sites from every treated sample are pooled into a single set and the same depth × ratio cutoff grid is applied.

What to look for:

  • The pooled count will always be >= the largest single-sample count, since different samples may cover different sites.
  • If the pooled count is much larger than any individual sample, samples are capturing largely non-overlapping sites — possibly due to different transcript coverage or biological replicates.
  • If pooled ≈ single-sample counts, samples are highly concordant in which sites they detect.

Depth distribution

overlap_all <- map_dfr(seq_len(nrow(cutoffs)), function(i) {

  d   <- cutoffs$min_depth[i]
  r   <- cutoffs$min_ratio[i]
  lab <- cutoffs$label[i]

  pooled_clean <- pooled_sites %>%
    filter(depth >= d, ratio >= r) %>%
    mutate(ref = as.integer(ref))

  pooled_keys <- pooled_clean %>%
    mutate(key = paste(ref, pos, sep = "_")) %>%
    pull(key) %>% unique()

  bind_rows(
    data.frame(key = others_keys, source = "data_others"),
    data.frame(key = pooled_keys, source = "pooled_sites")
  ) %>%
    group_by(key) %>%
    summarise(in_others = any(source == "data_others"),
              in_pooled = any(source == "pooled_sites"),
              .groups = "drop") %>%
    mutate(
      ref      = as.integer(str_extract(key, "^\\d+")),
      category = case_when(
        in_others & in_pooled ~ "Both",
        in_others             ~ "data_others only",
        TRUE                  ~ "pooled_sites only"
      ),
      cutoff    = lab,
      min_depth = d,
      min_ratio = r,
      n_pooled  = length(pooled_keys),
      n_others  = length(others_keys)
    )
})

Compute overlap across cutoffs

For each cutoff, every pooled site is tested against all published intervals on the same chromosome. A pooled site overlaps a published entry if its position falls within [pos_start, pos_end]. Conversely, a published entry is counted as overlapping if at least one pooled site falls within its interval.

find_overlap2 <- function(pooled_clean) {

  # for each pooled site: does it fall within any published interval?
  pooled_with_overlap <- pooled_clean %>%
    mutate(
      in_others = map2_lgl(ref, pos, function(r, p) {
        any(data_others2_clean$ref == r &
            data_others2_clean$pos_start <= p &
            data_others2_clean$pos_end   >= p)
      })
    )

  # for each published interval: does any pooled site fall within it?
  others_with_overlap <- data_others2_clean %>%
    mutate(
      in_pooled = pmap_lgl(list(ref, pos_start, pos_end), function(r, s, e) {
        any(pooled_clean$ref == r &
            pooled_clean$pos >= s &
            pooled_clean$pos <= e)
      })
    )

  list(pooled = pooled_with_overlap,
       others = others_with_overlap)
}
