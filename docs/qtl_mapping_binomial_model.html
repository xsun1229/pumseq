Overview

We map genetic effects on pseudouridylation (%pU) with the count-based model as a starting point: model the raw converted-read counts as binomial, test the genotype effect on the log-odds of the modification rate, and assess significance with a permutation LOD threshold.

Sites tested: the 5-fold union (called in ≥ 1 fold) set, 12,662 sites (see the 5-fold site-calling page). Samples: the 50 genotyped cell lines.

The model

For a given pU site, let individual \(i\) have \(y_i\) converted reads out of \(n_i\) total reads (\(n_i = C_i + T_i\), the read depth at that site):

\[ y_i \mid p_i \;\sim\; \text{Binomial}(n_i,\, p_i) \] \[ \text{logit}(p_i) \;=\; \beta_0 \;+\; \beta_g\, G_i \;+\; z_i^\top \gamma \]

  • \(p_i\) — the site’s true modification rate in individual \(i\).
  • \(n_i\) — read depth; enters as the binomial trial count, so low-depth observations automatically carry less weight.
  • \(G_i \in \{0,1,2\}\) — genotype of the test SNP (a cis variant near the site).
  • \(\beta_g\) — the genetic effect we test.
  • \(z_i\) — covariates (e.g. genotype PCs to correct population structure); optional in this first pass.

This is a standard binomial GLM (logit link). It is the simplest member of the count-model family — no overdispersion parameter, no random effects.

QTL mapping via LOD

For each (site, cis-SNP) pair:

  1. Fit the full model \(\text{logit}(p_i) = \beta_0 + \beta_g G_i (+ z_i^\top\gamma)\) and the null model with \(\beta_g = 0\).
  2. Compute the LOD score:

\[ \text{LOD} \;=\; \frac{\log\text{Lik}_{\text{full}} - \log\text{Lik}_{\text{null}}}{\log(10)} \]

Higher LOD = stronger evidence that the test SNP affects %pU at that site. For a site with several cis SNPs, the site’s signal is the maximum LOD over its SNPs.

Significance: permutation threshold

A genome-wide, multiple-testing-aware threshold (Churchill & Doerge 1994):

  1. Permute the phenotype-to-genotype assignment across individuals (breaks any real genotype-phenotype relationship), ~1000×.
  2. For each permutation, record the genome-wide maximum LOD across all (site, SNP) tests.
  3. The 95th percentile of those 1000 maxima is the genome-wide 5% significance threshold.
  4. Sites/SNPs whose observed LOD exceeds the threshold are candidate QTL.

Data

  • Sites: 5-fold union, ≥ 1 fold — 12,662 pU sites. (A stricter ≥ k-fold set could be substituted; see the 5-fold page https://xsun1229.github.io/pumseq/mRNA_5fold_union_site_calling.html.)
  • Phenotype: per site, per cell line — converted reads \(C\) (= \(y\)) out of total depth \(N\) (= \(C+T\)), from the pooled joint counts.
  • Genotypes: cis SNPs within a window of each site (e.g. ±5kb, 20kb, 50kb, 100kb), from the imputed genotypes of the 50 cell lines.
  • Covariates \(z\): phenotype PCs computed from M-values (see below).

Covariates: M-value phenotype PCs

We follow the approach used in the IMAGE paper (https://pubmed.ncbi.nlm.nih.gov/31651351/), which adjusts for the top methylation principal components computed from M-values.

How they are computed. For each site \(s\) and cell line \(i\), convert the counts to an M-value — the log-odds of the modification rate, with a pseudocount so that \(y = 0\) or \(y = n\) stay finite:

https://link.springer.com/article/10.1186/1471-2105-11-587

\[ M_{si} \;=\; \log_2 \frac{y_{si} + 1}{\,n_{si} - y_{si} + 1\,} \]

Then:

  1. Standardize each site across cell lines (centre and scale each row), so that no individual site dominates the components.
  2. PCA across individuals on the standardized M-value matrix.
  3. Include the top \(k\) PC scores (one value per cell line) as the covariates \(z_i\) in the binomial GLM.
M  <- log2((C + 1) / (N - C + 1))     # sites x cell lines
Ms <- t(scale(t(M)))                       # standardize each site
Ms[!is.finite(Ms)] <- 0
pc <- prcomp(t(Ms), center = TRUE, scale. = FALSE)
z  <- pc$x[, 1:k, drop = FALSE]            # cell lines x k covariates

How many PCs. Not fixed in advance — we will sweep \(k \in \{3, 5, 7, 10, 15, 20\}\) and choose based on the results, as was done for the earlier tensorQTL PC sweep.
